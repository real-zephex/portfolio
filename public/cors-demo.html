Direct cross-origin request

Your browser sends a request directly to a different origin. The browser enforces the Same-Origin Policy — if the server doesn't respond with the right Access-Control-Allow-Origin header, the response is blocked before your code ever sees it.

Request routed through a CORS proxy

A CORS proxy sits between your browser and the target API. Because it runs server-side, it's not bound by the browser's Same-Origin Policy. It forwards your request, gets the response, and adds Access-Control-Allow-Origin: * before sending it back.
